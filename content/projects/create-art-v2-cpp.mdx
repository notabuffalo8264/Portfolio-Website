## How This Started

Create Art V2 started after I spent time using my Java version of this project and realized I wanted a workflow that felt more practical for daily sketching. I rebuilt the idea in **C++** so I could push the app further and learn lower-level application design along the way.

After switching to a touchscreen laptop, supporting stylus input became a big priority. I wanted to keep the speed of keyboard-driven interaction from V1 while adding more natural pointer-based drawing.

## What it does

The app supports both **stylus input and keyboard controls**, making it flexible for quick sketches, notes, and explaining ideas. A collapsible side panel keeps tools available when needed without permanently taking over workspace.

Compared with V1, this version focused more on usability and control flow, with a cleaner layout and broader interaction options.

## Why I built it

After using the first version, I wanted something that felt better for real daily use, especially on a touchscreen laptop. Supporting stylus and keyboard input together made the app much more practical for how I actually sketch and explain ideas.

At the same time, this was a learning project for C++, so I intentionally did not use AI to generate the code. I used V2 as a way to deeply understand input handling, rendering, and state management by building it myself.

## Challenges & Limitations

As the feature set grew, performance started to degrade during longer drawing sessions. The biggest issue came from storing a full history of actions for undo support, which pushed memory usage up over time.

The app worked well for shorter sessions, but this limitation highlighted the tradeoffs involved in custom state/history systems and where optimization work is still needed.

## Looking Ahead

This project is part of an ongoing idea rather than a finished endpoint. It helped define what worked from V1, what needed redesign, and what architectural decisions matter most when scaling an interactive drawing app.
